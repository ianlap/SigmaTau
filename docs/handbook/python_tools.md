# Python Tools Reference

These tools focus on plotting, interactive fitting, and cross-validation reporting.

## `plot_devs.py`

**Purpose:** Generate log-log stability plots for deviation outputs.

**Syntax:**
```bash
python3 scripts/python/plot_devs.py <dataset>
```

## `plot_kf.py`

**Purpose:** Generate multi-panel KF diagnostics.

**Syntax:**
```bash
python3 scripts/python/plot_kf.py <dataset>
```

## `plot_mhdev_preview.py`

**Purpose:** Plot preview output for noise characterization.

**Syntax:**
```bash
python3 scripts/python/plot_mhdev_preview.py <dataset>
```

## `mhdev_fit_interactive.py`

**Purpose:** Interactive fit tool for power-law noise components.

**Syntax:**
```bash
python3 scripts/python/mhdev_fit_interactive.py <dataset>
```

## `generate_comprehensive_report.py`

**Purpose:** Build side-by-side validation report (Stable32 / allantools / SigmaTau).

**Syntax:**
```bash
python3 scripts/python/generate_comprehensive_report.py
```

## Input/Output expectations

- Inputs are generated by Julia/MATLAB pipeline scripts or validation datasets.
- Outputs are figures/reports saved in dataset-specific output folders.

## Troubleshooting

- Use a Python environment with `numpy`, `matplotlib`, `pandas` (and project-specific deps as needed).
- If plots are blank/missing, verify prerequisite pipeline outputs exist for the dataset label.
