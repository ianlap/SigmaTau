# SigmaTau — Gemini Critical Review & Refactor Plan (2026-04-14)

## 1. Performance & Memory Optimization (High Priority)

### Julia Kernels (`mtotdev`, `htotdev`, `mhtotdev`)
- **Observation**: Current implementations in `julia/src/deviations.jl` allocate multiple temporary vectors (`seq`, `seq_det`, `ext`, `cs`) inside the `nsubs` loop. This causes significant GC pressure for large datasets.
- **Action**: 
    - Move all buffer allocations outside the `nsubs` loop.
    - Use `copyto!` and in-place arithmetic (e.g., `.=`, `map!`).
    - Standardize the `detrend` calls to avoid unnecessary vector copying.

### Julia Kalman Filter (`kalman_filter`)
- **Observation**: `P_history` is currently a `Vector{Matrix{Float64}}`, leading to $N$ separate allocations for $3 \times 3$ or $5 \times 5$ matrices.
- **Action**: 
    - Refactor `P_history` to be a 3D array: `Array{Float64, 3}(undef, ns, ns, N)`.
    - This reduces $N$ allocations to a single allocation and improves cache locality for downstream analysis.

### MATLAB Total Deviation Kernels
- **Observation**: Loops perform repeated array concatenations (`xstar = [...]`) and slice copies.
- **Action**: 
    - Use pre-allocated buffers where possible.
    - Use direct indexing to access reflected elements instead of building a $3N$ length `xstar` array in memory.

## 3. Numerical Integrity & Standards

### TOTDEV Denominator — RESOLVED 2026-04-14
- **Resolution**: Verified against SP1065 §5.2.11 PDF. Eq. 25 (phase form) specifies
  `2τ²(N-2)`; Eq. 26 (frequency form) specifies `2(M-1)` with M = N-1 (self-consistent
  since M-1 = N-2). Current implementation `2(N-2)(mτ₀)²` is correct and matches the
  spec. The prior claim that SP1065 uses `2(N-1)(mτ₀)²` was incorrect — do not change.

### Confidence Intervals (CI)
- **Observation**: Verify that `ci.m` and `ci.jl` are using identical chi-squared quantile calculations.
- **Action**: Standardize the fallback logic for low EDF (e.g., EDF < 1) to ensure consistent reporting.
