## CRITICAL — Likely incorrect, data-corrupting, or contradicts cited reference

1. **`matlab/+sigmatau/+stats/ci.m` (lines 52–56)** — Chi-squared CI silently falls back to a Gaussian approximation when `chi2inv` is unavailable (no Statistics Toolbox). Users get a less-accurate CI with no warning. Emit an explicit `warning(...)` on fallback, or document the dependency at the wrapper level.

2. **`matlab/+sigmatau/+dev/engine.m` (line 79)** — `sqrt(max(var_val, 0))` masks negative variance from catastrophic cancellation by silently clamping to 0, hiding a numeric failure behind a valid-looking "0 deviation." Return `NaN` (or raise) when `var_val < -eps * scale` so the failure is visible.

3. **`julia/src/stats.jl` (line 73)** and **`matlab/+sigmatau/+stats/calculate_edf.m` (line 16)** — Guard `alpha + 2*d <= 1 → NaN` is not tied to a specific SP1065 equation and conflates invalid-parameter with boundary-case. Cite the exact Greenhall equation for the validity range and decide explicitly whether the boundary is strict (<) or inclusive (≤).

4. **`julia/src/deviations.jl` (line 618)** and **`matlab/+sigmatau/+dev/mhtotdev.m` (line 38)** — `mhtotdev_kernel` calls `detrend_linear(phase_seg)` on a `3m+1`-length subsegment. For `m ≤ 0`, detrend silently no-ops. Engine should reject `m < 1` up front; kernel should assert.

5. **`matlab/+sigmatau/+stats/bias.m` (lines 42–45)** — MTOT bias uses `idx = round(alpha(k)) + 3`, which goes out-of-bounds for `|alpha| > 2`. If noise ID ever returns α=3 or α=-3 (empirical fits can diverge), MATLAB either throws or returns 1.0 silently. Clamp α to `{-2..2}` with a warning, or use a `containers.Map` with explicit default.

## HIGH — Architecture violations, dead code, parity risks

6. **`matlab/+sigmatau/+dev/engine.m` (lines 94–96)** — EDF dispatch keys off `params.name` via a helper function `total_type_for_name`. Adding a new deviation without updating that helper silently falls through to `calculate_edf` with wrong coefficients. Make the EDF path an explicit field on `DevParams` (e.g., `params.edf_method = :greenhall | :totaldev`) so wrappers declare their intent.

7. **`julia/test/test_total_family.jl` (line 78)** — `htotdev m=1 uses hdev` test uses `rtol=0.01` to paper over the fact that htotdev applies bias correction while hdev does not. Either disable bias correction for the comparison or explicitly verify the bias factor; a 1% tolerance buries the contract.

8. **`matlab/+sigmatau/+noise/identify.m` (lines 125–203)** — `identify_b1rn` reimplements AVAR and MDEV via local `simple_avar` / `simple_mdev` helpers. This duplicates engine kernels and violates the "shared engine" architecture from CLAUDE.md. Call `sigmatau.dev.adev`/`mdev` or extract the kernels to a shared internal module.

9. **`matlab/+sigmatau/+kf/optimize.m` (line 207)** — `MAX_LS_SAMPLES = 100` is a magic number with no reference. First 100 samples can contain transients. Expose as `cfg.ls_init_samples` with a documented default, and cite the rationale.

10. **MATLAB ↔ Julia parity: NaN→0 rounding** — `matlab/+sigmatau/+dev/engine.m` (100–106) handles `alpha_float` with explicit `isnan` loop; `julia/src/engine.jl` (~line 87) uses a terse comprehension `[isnan(a) ? 0 : round(Int, a) for a in alpha_float]`. Functionally equivalent, stylistically divergent. Add a comment on the Julia side documenting the NaN→0 substitution and why 0 (white-FM) is the safe default.

11. **`julia/src/deviations.jl` (line 444)** — Loop bound written as `for j in 0:(6m - 3m)` instead of `0:3m`. Just simplify.

12. **`tdev`/`ldev` wrappers (MATLAB line 20; Julia line 136)** — Both scale CI by the TDEV/LDEV factor after bias correction has been applied inside the engine. The wrappers quietly assume no upstream bias correction; if the engine order changes, CI scaling silently breaks. Add a docstring contract on both wrappers stating "assumes no bias correction applied to the underlying mdev/mhdev variance."

13. **`matlab/+sigmatau/+stats/calculate_edf.m` (104 lines)**, **`matlab/+sigmatau/+noise/identify.m` (268 lines)**, **`matlab/+sigmatau/+kf/optimize.m` (208 lines)** — All exceed the 100-line CLAUDE.md limit. Extract nested helpers to sibling files (e.g., `+stats/private/greenhall_sz.m`, `+kf/private/kf_nll.m`, `+noise/private/b1_ratio.m`).
